# Use one versioned Blackboard v2 Project Interface

Blackboard v2 exposes one closed semantic contract through six trusted tools—change, current read, history, Evidence retention, Attempt checkpoint, and finish—and path-versioned `/api/v2` HTTP adapters with the same service semantics. HTTP uses honest status codes, bearer/daemon authentication, `Idempotency-Key`, revision ETags, and cursor-paginated history; Runtime MCP payloads never accept Project, Task, Continuation, or origin identity. A common optional synchronization attachment lets the next authenticated trusted response carry the complete new Runtime Blackboard Snapshot even when the attempted semantic operation fails validation. This replaces the graph-v1 tool/route catalog instead of maintaining compatibility aliases, trading a deliberate breaking cutover for one small, predictable interface that can evolve only additively within v2.

模块由两个子包组成，职责清晰分层：
- `internal/modelprovider`：核心领域层。`modelprovider.go` 定义 `Provider`/`Endpoint`/`Catalog` 等数据模型与 `Service` CRUD，通过 `store.DB` 直接以 SQLite JSON 列 (`protocols_json`/`endpoints_json`/`catalog_json`) 持久化；`resolver.go` 暴露 `Resolve(ResolveRequest) (Snapshot, error)`，依赖 `runtimeplugin.Registry`、`credential.Service`、`runtimeprofile.Profile` 将 profile 中的 provider_id + protocol + model 解析为可注入的 Snapshot，并校验 API Key 是否可通过环境变量或 credential binding 获取。
- `internal/modelprovidermigrate`：一次性迁移工具层。`Service.Preview`/`Apply` 读取旧版 `runtimeprofile.Fields`（`endpoint`/`model`/`api_keys`/`env`），基于 `runtimeplugin.Plugin.ModelProvider.SupportedProtocols` 调用 `modelprovider.BackfillEndpoints` 生成 `ProposedProvider`，支持 `create`/`reuse` 两种 Action，并在 Apply 后调用 `ClearLegacyModelFields` 清理已迁移的旧字段。

依赖方向：`modelprovidermigrate` → `modelprovider` → `runtimeplugin`/`runtimeprofile`/`credential`；`modelprovider` 不反向依赖迁移包。对外契约集中在 `Service` 方法签名、`ResolveRequest`/`Snapshot`、`Action`/`Preview`/`ApplyRequest` 结构体上，供 daemon_server 与 CLI 消费。

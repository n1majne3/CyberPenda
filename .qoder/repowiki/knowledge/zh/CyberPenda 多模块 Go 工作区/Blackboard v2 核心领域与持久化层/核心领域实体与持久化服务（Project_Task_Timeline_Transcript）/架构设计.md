模块按领域拆分为四个独立包，每个包遵循「类型 + Service + 错误」的固定结构：
- `internal/project`：Project/Scope/Defaults 模型及 Service，负责工程边界与默认值 CRUD，通过 `store.DB` 直接写 SQLite。
- `internal/task`：Task/Event/TaskContinuation/RuntimeConfigVersion 等模型及 Service，维护任务生命周期、事件追加（含 seq 单调递增事务）、Continuation 启动/替换/终端状态协调；依赖 `project.Service` 仅读取 scope 快照，反向不依赖 task。
- `internal/timeline`：纯投影函数 `Build([]task.Event) []Item`，将任务事件聚合成多轮 thinking/tool_use/tool_result 时间线条目，依赖 `runtimeoutput` 解析器。
- `internal/transcript`：纯投影函数 `Build(task.Task, []task.Event) []Entry`，生成带 role/kind/stream 的可读对话记录，并基于 `runtimeplugin.Registry` 选择适配器解析器。

依赖方向单向：timeline → task → project；transcript → task + runtimeplugin；两个投影包不持有任何持久化句柄，仅消费 `task.Event` 切片。所有 SQL 使用 `?` 参数化与 `time.RFC3339Nano` 序列化，复杂字段以 JSON 列存储并通过 `scanXxx` 辅助函数反序列化。

本模块是 `internal/runtimeplugin` 与 `internal/runtimeprofile` 之上的纯函数层，不持有状态也不实现 `runtime.Adapter` 接口，从而允许 Slice 9 验收用例在无真实二进制时运行。
- 入口函数集中在 `adapters.go`：`BuildLaunchArgs` / `BuildNativeResumeArgs` 通过 `runtimeplugin.MustBuiltinRegistry()` 查表得到 Provider 插件的 LaunchTemplate，再经 `RenderLaunch` 渲染 argv；`DetectBinary` 仅做存在性与可执行性检查（自定义轻量 `lookPath` 避免引入 `os/exec`）；`Redactor` 基于正则形状 + 可选已知值精确匹配对事件载荷脱敏。
- `blackboard_v2_resume.go` 单独承载 Blackboard v2 语义：`BuildBlackboardV2ResumePrompt` 仅拼接 Task Goal、未消费 Harness Steering 与中断 Checkpoint，把“语义状态”与“快照快照”解耦。
- 依赖方向单向：本模块 → `runtimeplugin` / `runtimeprofile` / `blackboardv2`，不被 runtime 子模块反向依赖，保证可被测试层直接引用。

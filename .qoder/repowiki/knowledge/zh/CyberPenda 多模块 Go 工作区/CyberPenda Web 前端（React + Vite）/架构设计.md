模块为独立的前端应用，入口 `src/main.tsx` 通过 `createRoot` 挂载 `<ThemeProvider><App/>`；路由由 `src/App.tsx` 中的 `createBrowserRouter` 集中声明，外层 `ShellLayout` 提供侧边栏/移动端抽屉布局与 `Outlet`，子路由按 `/projects/:projectId/*` 组织到各 Page 组件。

分层结构：
- `pages/`：每个路由对应一个 `.tsx` 页面文件，附带同名的 `*.test.tsx` 测试文件以及若干 `*.ts` 表单/视图辅助模块（如 `taskLaunchForm.ts`、`modelProviderForm.ts`），页面只负责 UI 编排与状态展示。
- `components/`：跨页面复用的无状态/上下文组件（Logo、PageHeader、ProjectPageShell、SaveActionButton、ThemeProvider、ui.tsx 等），主题通过 `theme-context.ts` + `ThemeProvider.tsx` 注入。
- `lib/`：领域无关的基础库——`api.ts` 是统一的 typed HTTP 客户端（封装 `fetch`、Bearer token 注入、`ApiError` 抛出），`blackboardv2.ts` 实现 Blackboard v2 协议的严格解析器（`parseRuntimeSnapshot` / `parseCurrentDetail` / `parseSemanticHistory` / `parseSemanticHealth`），使用白名单字段校验与 `require*` 断言函数保证后端契约；`format.ts`、`utils.ts` 提供通用工具。
- `test/`：全局 Vitest 环境（`setup.ts`、`mockApi.ts`）。

构建与集成：`vite.config.ts` 配置 `@` 路径别名、开发代理 `/api`、`/health` → Go daemon `:8787`，生产构建产物输出至 `dist/` 供 daemon 嵌入；`internal/daemon/webfs/dist/` 是 daemon 侧的 webfs 镜像目录。依赖方面使用 lucide-react 图标、class-variance-authority + clsx + tailwind-merge 组合样式，字体通过 @fontsource/geist 静态引入。

根目录作为 Go workspace 的单一入口：go.mod 声明 module pentest 并集中锁定 sqlite、MCP SDK、goldmark 等跨子模块共享依赖；Makefile 提供 dev/build/test/smoke 等顶层命令，将 cmd/pentestd（daemon_server）与 web/frontend 通过 Vite proxy 在 :8787 上联调，并将 React 产物 rsync 进 internal/daemon/webfs/dist 由 daemon embed。

子模块间通过以下契约耦合而非直接 import：
- Blackboard v2 Contract（JSON Schema / OpenAPI / 关系语法）由 blackboard_v2 冻结，core_domain、pentestctl_cli、web_frontend 均按同一份契约读写，确保语义一致。
- MCP tool schema 由 daemon_server 暴露，runtime_sandbox 内的 runner 通过 MCP 协议调用，形成“沙箱 → MCP → daemon”的单向调用链。
- Provider Session 接口由 model_provider 定义，daemon_server 与 runtime_sandbox 分别作为配置方与执行方消费该接口。
- Skills System 的 SKILL.md 指令文档被 skills_system 持久化后，由 daemon_server 的 Skill 处理器注入到任务 Profile，供 runtime_sandbox 读取。

三个顶级模块（blackboard_v2、daemon_server、runtime_sandbox）各自内部子模块边界稳定，外部仅通过上述契约交互，根级不引入额外业务逻辑。

模块以 `runtimeprofile.go` 为中心定义核心类型（`Profile`、`Fields`、`Provider`、`MCPServer`、`RuntimeExtensionRef`）和 `Service` 领域服务：通过 `store.DB` 直接读写 SQLite 表 `runtime_profiles`，字段序列化为 `fields_json` JSON 文本；对外暴露 Create/List/Get/Update/ReplaceFields/Delete/PromoteToPreset 等 CRUD 方法，并附带 `GeneratedConfig` 只读预览。

- 验证与归一化分层：`custom_args_conflict.go` 按 Provider 维护 flag/config-key → 结构化字段的冲突规则表，拒绝在 Custom Args 中重定义 model / model_provider / reasoning_effort；`reasoning_effort.go` 集中管理 ReasoningEffort 枚举及三层优先级解析（Turn Selection > Launch Override > Profile Default > high）；`apikeys.go` 提供密钥脱敏、合并与物化工具。
- 启动解析层：`launch.go` 定义最小化的 `LaunchSelection`，提供 `MatchesLaunch`/`FindLaunchProfile` 匹配函数与 `ResolveLaunchProfile` 懒创建逻辑，当不存在匹配 Profile 时以 `ProfileKindLaunchResolve` 自动生成仅含 ModelProviderID/ModelOverride/DefaultRunner 的最小 Profile。
- 依赖方向单向：本模块仅依赖上层 `pentest/internal/store` 获取 DB 句柄，不反向依赖 daemon_server 或 runner_integration，后者通过 Service 接口消费 Profile 数据。

本模块是 runner 子包中的纯函数式投影层，入口为 `ProjectRuntimeConfig`，按 `runtimeplugin.Plugin.ConfigProjection.Primitive` 分派到 provider-specific 投影器：`projectClaudeSettings`（settings.json + .mcp.json）、`projectCodexConfig`（config.toml + auth.json）、`projectPiConfig`（agent/models.json + agent/auth.json），其余走 `projectGenericConfig` 生成通用 config.json。

- 输入聚合于 `ProjectionRequest`，包含 ProjectID/TaskID、`credential.Service`、`modelprovider.ProviderGetter`、`runtimeplugin.Registry`、`runtimeextension.Registry`、预捕获的 `GlobalModelProviderSnapshot` 以及可选的 `MaterializedCredentials` 内存快照；输出为 `ConfigProjection`（含 ConfigPath、预览用 Config map、ResolvedProfile、ModelSnapshot）。
- 凭据解析遵循单一优先级链：`MaterializedCredentials` → `ModelSnapshot.APIKeyEnv` 环境变量 → profile inline keys → `credential.Service.ResolveMaterializedEnv`，并通过 `secretEnvKeyPattern` 在 preview/redact 路径统一脱敏。
- Pi 全局模型集通过 `piLaunchReadyProvider` 过滤出 launch-ready 的全局 Provider，写入 models.json 并在 `mergePiProjectedCredentials` 中合并其 API Key，使跨 Provider 轮次无需重启；`CloneGlobalModelProviderSnapshot` 提供不可变快照以规避 Store TX 死锁。
- 技能包与运行时扩展分别经 `projectSkillBundles` 与 `projectRuntimeExtensions` 复制到 `layout.SkillsRoot` / `layout.ProviderHome`，前者校验 bundle 元数据，后者仅允许 `local_dir`/`local_file` 源且禁止符号链接。
- 进程环境由 `LaunchProcessEnv` / `LaunchProcessEnvWithCredentials` 组装，优先渲染插件 manifest 的 `ProcessEnv`，再叠加 profile env 与 materialized 凭据；`LaunchConfigPath` 将宿主路径映射为 `/task/...` 容器路径。
- 依赖方向单向向外：仅消费 `internal/credential`、`internal/modelprovider`、`internal/project`、`internal/runtimeextension`、`internal/runtimeplugin`、`internal/runtimeprofile`、`internal/skill`，不反向引用 runner 内部其他组件。

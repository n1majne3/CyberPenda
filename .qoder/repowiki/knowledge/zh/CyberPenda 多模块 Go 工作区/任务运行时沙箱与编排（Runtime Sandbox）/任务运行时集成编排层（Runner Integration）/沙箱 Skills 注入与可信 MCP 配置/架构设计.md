本模块位于 `internal/runner` 包内，承担两类面向运行时的“投影”职责：
- Skills 注入（sandbox_skills.go）：根据 `runtimeprofile.Provider` 选择目标路径（Claude Code → `.claude/skills`，Codex/Pi → `.agents/skills`），通过 `symlinkUnlessExists` 将镜像内置的 `/opt/pentest/skills` 以符号链接形式挂载到工作目录与 Provider Home，避免复制。
- 可信 MCP 配置（mcp.go）：从 `ProjectionRequest` + `Profile` 组装 `TaskContext`，计算可达的 API/MCP URL（区分 sandbox 使用 `host.docker.internal`），按 `PENTEST_DISABLE_TRUSTED_MCP` 环境变量决定是否注入名为 `pentest` 的可信服务器；对 Claude Code 额外输出 `.mcp.json`，对 Pi 输出 `agent/mcp.json`，对 Codex 追加 TOML 片段；同时写入 `.pentest/context.json`、`.pentest/scope.json` 以及根级 `AGENTS.md` 作为运行时调用约定说明。
依赖方向单向指向 `runtimeprofile`（Provider、MCPServer、Profile）与 `project.Scope`，不反向依赖 daemon 或具体 HTTP 客户端，仅产出路径与配置文件供上层编排进程消费。

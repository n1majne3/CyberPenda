本模块是 `internal/runner` 顶层编排入口，对外暴露三类契约：
- `Layout` / `PrepareTaskLayout`：定义 task_root/workdir/runtime-home/provider_home/skills/artifacts/logs 目录结构，由 `blackboard_v2` 子模块校验其一致性。
- `ConfigProjection` / `ProjectRuntimeConfig`：通过 `projection_layer` 子模块按 Provider 分发到 `projectClaudeSettings`、`projectCodexConfig`、`projectPiConfig`，写入 provider_home 下的 settings/config/auth 等文件，并注入 skills 符号链接与可信 MCP 服务器列表（`sandbox_skills_and_mcp`）。
- `Command` / `BuildSandboxCommand`：基于 `Layout` + `SandboxNetworkMode` 生成只读 bind mount 的 Docker create 参数，不执行容器；`target.go` 提供 `RewriteLoopbackTargets` 把 127.0.0.1/localhost 改写为 `host.docker.internal` 以打通宿主机网络。

跨子模块协作的关键点：
- `GlobalModelProviderSnapshot` 作为不可变快照在 `ProjectionRequest` 中传递，避免 Pi 全局 Provider 投影进入 Store 事务导致预提交竞态。
- `providerHomeDir` / `providerHomeEnv` 集中维护 claude/codex/pi 三者的 home 路径与环境变量映射，所有子模块共享同一份约定。
- 所有含敏感内容的 JSON 配置统一经 `writeJSONConfigFile(0o600)` 落盘，预览输出走 `redact*` 系列函数脱敏。

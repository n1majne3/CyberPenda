- 所有跨进程请求使用 `sha256.Sum256(完整编码帧)` 作为指纹，配合 pending/completed 两表实现幂等重试与冲突检测。
- 错误类型成对出现：`ErrXxxClosed/Started/Invalid/Mismatch` 常量 + 同名 `*Error` 结构体，并在 `Is` 中聚合等价错误以便上层统一处理。
- Provider 差异通过 `providerWireMethods` 函数指针声明式注入（send/interrupt/steer/permission/params/prepareSend/turnID/sessionID），避免在核心循环中分支判断具体 Provider。
- 异步通知与响应分离：带 ID 的消息走 Send 返回，无 ID 消息经 `ProtocolEmit` 回调由 `HandleEvent` 归一化为 `task.EventKindLifecycle/Steering/RuntimeOutput`。
- 资源清理使用 `sync.Once` + `Closed()/Terminated()` 双 channel 区分显式 Close 与意外退出，确保 `CloseAll` 时不会重复删除已退出的容器/进程组。

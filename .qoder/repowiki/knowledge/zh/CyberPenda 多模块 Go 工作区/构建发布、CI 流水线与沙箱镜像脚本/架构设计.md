本模块按职责分为三层：
- GitHub Actions 工作流（`.github/workflows/`）：`ci.yml` 触发 Go/Web/App Image 构建与按需 sandbox MCP smoke；`release.yml` 在 `v*` tag 上并行构建 release 二进制并发布到 GHCR + GitHub Release；`publish-sandbox.yml` 通过 matrix 分别在 amd64/arm64 原生 runner 上 push-by-digest，再合并为多架构 manifest；`smoke-runtime-nightly.yml` 手动触发带 LLM 密钥的 runtime smoke。
- 容器镜像（`docker/`）：`pentestd/Dockerfile` 使用 node:20 → golang:1.25 → alpine 三阶段构建，将 web/dist 嵌入 `internal/daemon/webfs/dist`；`pentest-sandbox/Dockerfile` 基于 kali-rolling，预装 ProjectDiscovery/Cairn 工具链、agent-browser、Claude/Pi/Codex CLI，并将 provider bridge 源码放在最后 COPY 层以复用缓存。
- 可执行脚本（`scripts/`）：`build-release-binaries.sh` 用 GOOS/GOARCH 循环交叉编译 pentestd/pentestctl 并生成 SHA256SUMS；`with-pentestd-live.sh` 启动临时 pentestd 进程并等待 /health 后再 exec 传入命令；`smoke-sandbox-mcp-live.sh` 从 sandbox 镜像内调用 Blackboard v2 `/api/v2/projects/{id}/blackboard/*` 边界；`run-juice-shop-live.py` 与 `smoke-runtime-tasks-live.py` 通过 HTTP API 创建 project/task 并轮询 fact 完成；`ensure-web-deps.sh` 检测 Vite/Rolldown 绑定缺失时自动 `npm ci`；`check-embedded-ui-sync.sh` 配合 `.githooks/pre-push` 阻止 stale UI 提交；`ci-sandbox-smoke-required.sh` 根据 diff 输出 `required=true/false` 供 CI 条件分支。
- 契约验证测试（`*_test.go`）：Go 测试读取 Dockerfile/Workflow/Makefile/README 等文本断言关键约束（如默认 sandbox 镜像统一为 `ghcr.io/n1majne3/cyberpenda-sandbox:latest`、release workflow 不得包含 sandbox 构建步骤、Blackboard v2 工具名集合等），形成“配置即测试”的自守契约。

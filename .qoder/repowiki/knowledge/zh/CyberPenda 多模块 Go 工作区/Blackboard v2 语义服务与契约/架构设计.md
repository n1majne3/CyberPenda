模块由三个相互协作的子包构成：
- `internal/blackboardv2`：核心语义服务层，以 `Service` 结构体暴露 `Apply`/`ApplyForContinuation`/`ReconcileTerminalContinuation`/`ProjectSemanticHealth` 等 API；通过 `ChangeBatch` + 多态 `Change`（op=create/update/relate/unrelate/transition/supersede/merge）在事务中写入 `blackboard_v2_records`、`blackboard_v2_relationships`、`blackboard_v2_attempt_origins`、`blackboard_v2_key_redirects` 等表，并通过 `WorkingSnapshot` 指针驱动 Runtime 侧 `.pentest/blackboard.json` 的增量同步。依赖方向单向指向 `store.DB` 与 `blackboardv2grammar`。
- `internal/blackboardv2contract`：只读契约包，使用 `//go:embed contractdata` 内嵌 `manifest.json`、`schemas/blackboard-v2.schema.json`、`openapi.json`、`relationships.json`、`trusted-tools.json` 及 golden fixtures，提供 `Harness` 用于校验任意 producer/adapter 输出；`TrustedTools()` 返回六个受信任 Runtime tool 及其输入/结果 schema 绑定，供 MCP tool 列表使用。
- `internal/blackboardv2grammar`：纯数据包，声明 11 个关系词表（about/part_of/tests/produced/evidences/supports/contradicts/derived_from/depends_on/satisfies/supersedes）、端点矩阵与 reason/self-link/cycle 策略，被 health 诊断与 Apply 路径共同消费。

文件按职责拆分：`service.go` 定义 DTO 与主入口，`change_batch_history_service_test.go` 等测试文件对应 `attempt_final_batch_service.go`、`checkpoint.go`、`continuity.go`、`evidence.go`、`finding.go`、`finish.go`、`health.go`、`json_decode.go`、`merge.go`、`projection.go`、`relationship_read_internal_test.go`、`replacement_proof_test.go`、`report.go`、`resume.go`、`retirement_test.go`、`solution.go`、`synchronization_test.go` 等子能力；`evidence_lock_unix.go` / `evidence_lock_windows.go` 按平台实现证据文件锁。所有 JSON 入参统一经 `decodeJSON` 做 UTF-8 合法性检查后再交给 encoding/json，避免 U+FFFD 吞掉语义差异。

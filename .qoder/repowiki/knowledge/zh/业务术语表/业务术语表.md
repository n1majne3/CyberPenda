### Project
- 定义：一个有边界的授权安全测试参与，拥有独立的 Scope、Task、记忆（Blackboard）、证据与报告。避免与 workspace/conversation/campaign 混用。

### Scope
- 定义：定义 Project 内授权测试资产边界与测试限制的范围声明。Scope Expansion 是新增资产或测试权限的提议，Out-of-Scope Fact 是超出当前范围但仍保留上下文的断言。
- 别名：Scope Snapshot

### Task
- 定义：由用户目标驱动的 Project 级运行，由一个 Runtime Profile 通过一个 Runner 执行。Task Goal 是用户的自然语言目标；Reason Task 是 operator 触发的规划型 Task，产出审批提案但不直接修改 Blackboard。
- 别名：Task Goal、Reason Task、Task Deletion、Task Finish

### Runtime
- 定义：被调度执行单个 Task 的本地代理 CLI 或助手进程。Task-Scoped Persistent Runtime 在一个 Task 的生命周期内跨 Runtime Turn 保持活跃，仅在显式完成、停止、失败或必需的重投影重启时关闭。
- 别名：Runtime Harness、Harness Steering、Runtime Continuation、Runtime Activity Indicator、Runtime Non-Interactive Defaults

### Runner
- 定义：Task 的运行时执行边界。Sandbox Runner 是默认选项，在隔离容器中运行；Host Runner 是显式 opt-in，直接在宿主机运行，必须在报告中可见且需 Operator 激活。Sandbox Runner 失败不会自动回退到 Host Runner。
- 别名：Sandbox Runner、Host Runner、Host Runner Activation

### Blackboard
- 定义：Project 本地的持久化语义记忆，包含 Entities、Exploration Objectives、Attempts、Project Facts、Findings、Solutions 与 Evidence Artifacts。Blackboard Key 是唯一稳定的语义标识符，Record Merge 将重复记录合并到一个规范记录并建立重定向。
- 别名：Blackboard Key、Record Merge、Blackboard Key Redirect、Semantic Change Batch、Semantic History、Current Work、Project Knowledge

### Entity
- 定义：Blackboard 中对“知识或探索工作关于什么”的持久身份，如主机、服务、端点、身份、文件或函数。其 scope status 描述记忆而不授予授权。

### Project Fact
- 定义：可跨 Task 复用的稳定 Project 级断言，具有 Fact Key 标识与版本历史。Tentative Fact 是尚未经确认的可行断言，Confirmed Fact 是有证据/重现/人工确认支撑的断言，Deprecated Fact 仍保留历史但不应视为当前事实。
- 别名：Fact Key、Fact Version、Deprecated Fact、Current Truth、Tentative Fact、Confirmed Fact

### Exploration Objective
- 定义：可复用的 Project 级探索方向，可能从现有 Project Facts、Findings 或 Solutions 推导而来，指向未知的未来结论。它不是 Current Truth 本身，但可被 Attempt 验证并最终通过 Project Facts、Findings 或 Solutions 解决。

### Attempt
- 定义：对 Exploration Objective、Entity、Project Fact、Finding 或 Solution 的一次探索试验的持久记录，以精炼结果收尾。Terminal Attempt 离开 Runtime 上下文前，所有正/负/阻塞/不确定结论必须以关联的 Project Fact、Finding、Solution、Evidence Artifact 或新 Objective 表示。

### Finding
- 定义：可报告的安全问题，包含严重性、证据、影响、建议与状态。CVSS Vector 用于派生严重性，CVSS Pending 表示已知问题形状但尚未提供完整向量。Finding Update 是部分更新，保留未指定字段。
- 别名：Finding Key、Finding Version、Finding Group、Confirmed Finding、CVSS Vector、CVSS Version、CVSS Pending、Finding Update

### Solution
- 定义：CTF Challenge 结论，表示候选、已验证、已拒绝或被替代的答案、flag 或程序。Solution 在 Pentest Project 中无效。Supersedes 关系用于标记替换与被替换记录。

### Evidence Artifact
- 定义：支持 Project Fact 或 Finding 的原始或衍生证明的持久引用。Artifact Root 是受管的项目或任务证据根目录，Task Artifact Root 是任务特定的证据根。
- 别名：Artifact Root、Task Artifact Root

### Trusted Origin
- 定义：Server 拥有的 Project 与执行绑定，用于验证谁或什么有权创建 Blackboard 变更或 Evidence Artifact。它是内部完整性数据，不是 Blackboard 知识或面向用户的审计内容。

### High-Risk Action
- 定义：可能导致中断、特权数据访问、认证影响、漏洞利用验证或其他超出普通枚举影响的测试动作。Intended Action 是高风险步骤前的预行动记录。Policy Violation 是记录的越权工作流违规。
- 别名：Intended Action、Policy Violation

### Reconciliation
- 定义：对被正常 Project Interface 写入之外发现的受管审查动作，可接受、拒绝或重新分类。Reconciliation Candidate 是未被信任的发现状态，必须由 operator 手动完成审查，不能自动完成。
- 别名：Reconciliation Candidate

### Report
- 定义：从 Findings、Project Facts、Blackboard Relationships 与 Evidence Artifacts 生成的交付物。

### Runtime Profile
- 定义：全局可编辑的配置，选择 Runtime 如何运行 Task，不存储密钥值。Launch-Resolved Runtime Profile 是通过 Launch Profile Resolution 创建的最小匹配 Profile，标记为 `launch_resolve`，可由用户提升为 Runtime Profile Preset。
- 别名：Launch-Resolved Runtime Profile、Runtime Profile Preset、Default Runtime Profile Preset、Launch Profile Resolution

### Model Provider
- 定义：全局可复用的非密钥模型服务配置，Runtime Profile 可在需要模型访问时使用。Model Provider ID 是不可变标识符，由其显示名生成并追加数字后缀以避免冲突。Model Catalog Refresh 通过 `/v1/models` 路径刷新模型列表。
- 别名：Model Provider ID、Model Provider Migration、Model Provider Endpoint、Model Catalog、Model Catalog Refresh、Model Override、Reasoning Effort

### Runtime Plugin
- 定义：声明式提供者定义，描述 Runtime Profile 如何启动、投影配置、验证启动与转录解析。内置插件包括 codex、claude_code、pi 与 fake。Plugin Manifest 是 JSON 配置文件，仅引用已知 Runtime Plugin Primitives。
- 别名：Runtime Plugin Manifest、Runtime Plugin Registry、Runtime Plugin Primitive

### Skill
- 定义：运行时无关的 Runtime Extension Bundle，通过 Skills Page 管理并可投影到任何支持的 Runtime。Built-in Skill 是由 daemon 从审核过的上游源播种的技能。Skill ID 是稳定标识符，Skill Source Provenance 记录来源与最后导入/编辑上下文。
- 别名：Built-in Skill、Skill Bundle Format、Skill Execution Boundary、Skill Deletion、Skill Preflight Preview、Task Skills Root

### Runtime Extension
- 定义：运行时原生插件、技能、包或配置包，由选定的 Runtime 在 Config Projection 后消费。Runtime Extension Library 是全局用户界面集合，Runtime Extension Import 是管理时的外部引入。Controlled Skill Import 是受控的包导入路径。
- 别名：Runtime Extension Bundle、Runtime-Specific Extension、Runtime Extension Library、Runtime Extension Import、Controlled Skill Import、Skill Publication、Skill Validation、Runtime Extension Enablement、Default Skill Enablement、Skill Opt-Out

### Credential Reference
- 定义：非密钥指针，让任务接收所需凭据而不在 Runtime Profile 中存储密钥。Credential Binding 是 Project 级的映射，Global Credential Binding 是默认映射，Disabled Credential Binding 明确阻止使用任何凭据源。
- 别名：Credential Binding、Global Credential Binding、Credential Binding Mode、Disabled Credential Binding

### Config Projection
- 定义：从 Runtime Profile、Model Provider 与 Credential References 进行任务本地运行时配置准备的过程。Generated Runtime Config 是可预览的任务本地配置输出。Profile Config Import 是将编辑后的运行时配置解析回结构化字段的操作。
- 别名：Generated Runtime Config、Profile Config Import

### Preflight
- 定义：记录的启动检查阶段，确定 Task 是否可以启动其 Runtime。Model Preflight Preview 显示解析的非密钥模型提供者投影与生成的 API Key 环境变量就绪情况。缺少 Model API Key Environment Variable 值或无法解析的 Credential Reference 都会阻止 Runtime 启动。
- 别名：Model Preflight Preview

### Project Interface
- 定义：支持的通道，让 Runtime 读写项目状态、记忆、证据与报告。CLI Fallback 是当主要代理集成不可用或不可靠时使用的命令行 Project Interface。直接存储更改需要在影响 Current Truth 或 Report 之前经过 Reconciliation。
- 别名：CLI Fallback

### Runtime Blackboard Snapshot
- 定义：当前主 Blackboard 图在某一修订的拓扑完整语义视图，包含每个当前可复用语义记录与关系的紧凑形式，排除辅助记录体、Trusted Origin 数据、审计历史与仅审计元数据。Launch Blackboard Pin 是 Immutable 的快照，Working Blackboard Snapshot 是任务本地可读的文件。
- 别名：Launch Blackboard Pin、Working Blackboard Snapshot、Blackboard Change Notice

### Attack Chain
- 定义：将 Project Facts 与 Findings 连接到可解释的安全测试故事的叙述路径。

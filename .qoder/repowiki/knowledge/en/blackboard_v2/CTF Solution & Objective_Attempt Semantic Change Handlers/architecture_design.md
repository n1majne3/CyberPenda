This leaf lives in `internal/blackboardv2` alongside the rest of the semantic change engine. `solution.go` contributes three handler functions (`applyCreateSolution`, `applyUpdateSolution`, `applySolutionTransition`) that plug into the central `Apply` dispatcher, plus a read-only `CTFSolvedState` query that derives a project's solved state from verified flag Solutions. Validation helpers (`completeSolutionRecord`, `partialSolutionRecord`, `validateSolutionRecord`, `ensureCTFProject`, `hasCurrentSolutionInvalidationMeaning`) enforce the CTF-only project kind guard, status/kind enums, immutability of verified solutions, and the requirement that rejection be backed by a contradicting Fact. Bidirectional adapters `recordFromSolution` / `Record.solutionRecord()` bridge between the domain-specific `SolutionRecord` and the generic `Record` used by the shared persistence layer. The two test files (`solution_service_test.go`, `objective_attempt_service_test.go`) are integration tests under package `blackboardv2_test` that exercise the same `Service.Apply`/`ApplyForContinuation`/`ReconcileContinuationAttempts` entry points against an in-memory SQLite store, asserting idempotency, version conflicts, authority boundaries across Continuations, and contract snapshots via `assertContractJSON`.

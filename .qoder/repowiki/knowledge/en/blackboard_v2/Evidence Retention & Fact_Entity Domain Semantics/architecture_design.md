Three cooperating files under package `blackboardv2`:
- `evidence.go` is the core: it defines the `RetainEvidenceRequest` closed JSON shape (with a custom `UnmarshalJSON` that rejects unknown fields), the `EvidenceConfig` (ArtifactRoot/RuntimeRoot/FailureInjector) injected into `Service`, and the full `RetainEvidenceForContinuation` pipeline — reservation (`INSERT OR IGNORE` on `blackboard_v2_evidence_requests`), managed vs. journaled temp payload verification, atomic publish via `ensureEvidencePublished`, graph commit, and replay from `blackboard_v2_idempotency_receipts`. It also contains all path planning helpers (`plannedEvidenceInternalPath`, `plannedEvidenceTempPath`, `semanticEvidencePath`) and secure file access through `os.OpenRoot` + `openSecureRegularFile` / `openExistingSecureDirectory` to prevent symlink escapes.
- `evidence_lock_unix.go` provides the build-tagged `lockEvidencePublisherFile` / `unlockAndCloseEvidencePublisher` using `syscall.Flock(LOCK_EX|LOCK_NONBLOCK)` for cross-process publication serialization; Windows has no implementation in this scope.
- `fact_service_test.go` is an end-to-end test suite exercising Fact/Entity create/update/clear/transition operations, idempotent replay, closed-shape validation before replay, confidence transition canonical ordering, project isolation, and the `supports` relation acyclicity check — all against an in-memory SQLite-backed `store` opened per test.

Dependency direction: `evidence.go` depends on `internal/store` for persistence and on shared helpers in the same package (e.g. `semanticError`, `decodeRequiredString`, `rejectUnknownFields`, `loadCurrentRecord`). The lock file is only imported by other blackboardv2 units at runtime; the test file imports `blackboardv2contract.Harness` to assert contract schemas.

- Every JSON envelope type implements a custom `UnmarshalJSON` that first decodes into `map[string]json.RawMessage`, rejects unknown fields, then validates required fields and schema strings before constructing the typed value — applied uniformly to `ChangeBatch`, `Change`, `CheckpointAttemptRequest`, and tuple types like `RecordVersionTuple`.
- Semantic errors are surfaced as a stable `Error` struct with `code`/`message`/`path`/`retryable`/`details` fields and constructed via a `semanticError` helper rather than returning plain `fmt.Errorf` values.
- Long-running write paths acquire `s.writeMu` at the top of `apply`/`SynchronizeContinuation`/`CreateContinuation` and defer unlock, ensuring only one batch or sync runs concurrently per Project.
- Crash-safe filesystem publication follows a publish-before-commit pattern: the Working Snapshot file is written under `runtimeRoot/<taskID>/workdir/.pentest/blackboard.json` before `tx.Commit()`, with a captured prior-byte backup restored via `restorePriorWorkingSnapshotFile` on any post-write error.
- Continuation-scoped APIs (`ApplyForContinuation`, `ReconcileTerminalContinuation`, `CheckpointAttemptForContinuation`, `AuthorizeContinuationBinding`) validate that `continuationID` is non-empty and resolve ownership via joins over `task_continuations`/`blackboard_v2_continuation_pins`/`blackboard_v2_continuation_state`, rejecting with `authority_denied` when the binding is missing or stale.

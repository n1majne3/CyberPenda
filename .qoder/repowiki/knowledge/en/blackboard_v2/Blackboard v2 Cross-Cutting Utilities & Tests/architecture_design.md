Three independent leaf files that do not depend on each other but augment the `blackboardv2` package:
- `json_decode.go` provides a private `decodeJSON` wrapper around `encoding/json.Unmarshal` that pre-validates input with `utf8.Valid`, returning a `semantic_validation` `Error` when bytes are invalid UTF-8 — this prevents Go's default U+FFFD replacement from collapsing distinct wire inputs into one semantic value.
- `evidence_lock_windows.go` is a build-tagged Windows-only implementation of file-level mutual exclusion for the evidence publisher, using `golang.org/x/sys/windows.LockFileEx` with an exclusive, fail-immediate lock over the full 32-bit range; it maps `ERROR_LOCK_VIOLATION` to a retryable `evidence_publication_in_progress` error so callers can back off instead of failing hard. A companion `unlockAndCloseEvidencePublisher` helper ensures unlock + close even on error paths.
- `change_batch_history_service_test.go` lives in package `blackboardv2_test` (external) and exercises the public `NewService`/`Apply`/`ReadHistory` surface against an in-memory SQLite-backed `store`, asserting frozen schema validation (`schema`/`idempotency_key`/`changes` presence), idempotent replay of empty batches, semantic-version ordering across relationship-history pages, and cursor staleness detection with structured `Details` fields.

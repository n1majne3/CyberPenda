Two cooperating files inside package `blackboardv2`:
- `finding.go` owns the server-side Finding model (`findingOutputRecord`) plus the create/update/patch pipeline (`applyCreateFinding`, `applyUpdateFinding`, `deriveFindingRecord`, `applyFindingPatch`). It enforces semantic invariants via `validateFindingOutputRecord`, derives `severity`/`cvss_pending` from CVSS v3.1/v4.0 vectors using `gocvss31`/`gocvss40`, and gates `confirmed` status on current `supports`/`evidences` relationships through `validateAllConfirmedFindings` + `findingHasCurrentSupport`. Terminal transitions (`false_positive`, `superseded`) require a `contradicts` meaning Fact, checked by `hasCurrentFindingInvalidationMeaning`.
- `report.go` defines closed consumer allowlists (`PentestReportProjection`, `CTFSolutionProjection`, `ReportFinding`, `ReportFact`, `ReportEvidence`) and builds them read-only from `blackboard_v2_records`/`blackboard_v2_relationships` in `ProjectPentestReport` / `ProjectCTFSolution`. Evidence is relationship-derived only: for Pentest reports it includes direct evidences on a Finding plus evidences of confirmed Facts that support it; for CTF it includes evidences linked to verified/candidate flags, answers, procedures, or confirmed Facts. Findings are sorted deterministically by severity → target → title → key.
- `finding_report_service_test.go` is an end-to-end test exercising both files together: change batches, evidence retention, cross-project isolation, rollback on broken support, supersession, and determinism of both JSON projections and downstream Markdown via `report.NewV2Generator` / `report.NewCTFV2Generator`.

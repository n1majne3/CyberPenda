Four independent `main` packages, each a single-process executable with no shared code:
- `cmd/pentestd/main.go` — HTTP server entrypoint; wires flags/env (`PENTEST_*`) into `daemon.NewServer`, starts an `http.Server`, and handles graceful shutdown via `signal.NotifyContext`. Delegates provider lifecycle to `daemon.ProductionProviderSessionFactory` which spawns the sandbox-local bridge.
- `cmd/pentestctl/main.go` — thin CLI shim delegating all logic to `pentest/internal/pentestctl.RunWithIO`; only responsible for exit-code mapping via `ExitCode(err)`.
- `cmd/pentest-provider-bridge/main.go` — Go process spawned inside the sandbox container; validates `-provider codex|pi`, execs the real provider binary, and translates its native line-delimited JSON protocol into CyberPenda's internal JSON-RPC envelope. The `piWire` struct is a stateful translator (session/turn tracking, request-id correlation) while the Codex path is a transparent `copyJSONLines` passthrough.
- `cmd/pentest-claude-sdk-bridge/{main.mjs,bridge.mjs}` — Node.js process wrapping `@anthropic-ai/claude-agent-sdk`; `main.mjs` parses `--model/--settings/--cwd/--resume` and injects the SDK's `query` function, while `bridge.mjs` implements the same JSON-RPC surface (`claude/initialize`, `claude/input`, `claude/interrupt`, `claude/permission/respond`) over stdin/stdout, exposing permission prompts and turn events as `claude/*` notifications.

Dependency direction: these four executables depend inward on `pentest/internal/*` (Go) or on published npm packages (`@anthropic-ai/claude-agent-sdk`); they contain no business logic themselves and exist solely to wire configuration, I/O, and process lifecycles.

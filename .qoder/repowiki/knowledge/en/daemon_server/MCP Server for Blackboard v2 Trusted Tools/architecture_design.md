Two-layer split:
- `internal/daemon/mcp_handlers.go` mounts a single `sdkmcp.NewStreamableHTTPHandler` on `/mcp`, resolves a Bearer token against `projectinterfaceGrants` to obtain a `Grant` (or a prebuilt `GrantError`), and injects it into `mcpserver.Deps`. It also disables localhost Host-header protection so sandbox runtimes reaching via `host.docker.internal` are accepted.
- `internal/mcpserver/v2.go` owns the MCP server lifecycle. `New(deps)` builds an `sdkmcp.Server` named `pentest-agent` and delegates tool registration to `registerBlackboardV2Tools`, which loads the frozen contract via `blackboardv2contract.NewHarness().TrustedTools()` and registers exactly six handlers (`blackboard_change`, `blackboard_read`, `blackboard_history`, `blackboard_retain_evidence`, `blackboard_checkpoint_attempt`, `blackboard_finish`). Each handler decodes arguments through `decodeV2ToolArgs` against the closed schema, then dispatches to `deps.callV2WithFingerprint`, which runs `requireGrant` + `AuthorizeContinuationBinding`, optionally claims/captures synchronization attachments, and marshals results as either plain JSON or an `{error,sync}` envelope via `toolBlackboardV2JSON` / `toolBlackboardV2Error`. The dependency direction is strictly daemon → mcpserver → blackboardv2/blackboardv2contract/projectinterface; the test in `mcp_test.go` exercises the HTTP surface end-to-end (health, Docker-host header acceptance, and exact tool inventory).

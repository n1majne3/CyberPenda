Two concerns live in package `daemon`:
- `modelprovider_handlers.go` / `modelprovider_migration_handlers.go` expose REST endpoints on `Server` that delegate to `server.modelProviders` (`internal/modelprovider`) and `server.modelProviderMigrator()` (`internal/modelprovidermigrate.Service`). Errors are mapped via `writeModelProviderError`/`writeModelProviderMigrationError` switch blocks over sentinel errors (`ErrNotFound`, `ErrMissingName`, `ErrInUse`, …) into standard HTTP status codes; credentials for provider API keys are materialized through `server.creds.Resolve` + `credential.Materialize`.
- `production_provider_session_factory.go` implements `ProductionProviderSessionFactory`, a non-PTY bridge assembly that opens long-lived sessions for three providers (Codex App Server, Claude Agent SDK bridge, Pi headless RPC) on both `task.RunnerSandbox` (Docker container) and `task.RunnerHost` (local process). It owns a `bounds map[string]ProviderSessionBinding` keyed by Task ID with a `sync.Mutex` to deduplicate rebinds via `rebindPrior`. Each provider path builds its argv through dedicated helpers (`hostClaudeSDKBridgeArgs`, `hostPiCustomArgs`, `hostCodexCustomArgs`) that drop structured launch-template flags while preserving user Custom Args, then delegates transport to `runtime.SandboxSessionBridgeRegistry` / `runtime.HostSessionBridgeRegistry` and wraps the resulting `runtime.ProviderSession` in `productionBoundSession` before handing it back as a `ProviderSessionBinding`.

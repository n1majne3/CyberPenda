Three-file leaf in package `daemon`:
- `server.go` defines `Config`/`Server`, owns `NewServer` which opens `store.DB`, builds `project.Service`, `runtimeprofile.Service`, `modelprovider.Service`, `skill.Service`, `credential.Service`, `task.Service`, `preflight.Service`, `runtime.Harness`, `blackboardv2.Service` (with evidence config), and a `projectinterface.GrantStore`; then calls `routes()` to register every `/api/*`, `/mcp`, and SPA handler on an `http.ServeMux`. `ServeHTTP` is the single gateway: it rejects DNS-rebinding via `allowedOrigin` (loopback + `host.docker.internal` + same listen addr), applies Bearer-token or query-string auth when `AuthToken` is set (skipping public paths like `/health`, `/assets/*`, `/index.html`), wraps the response with `statusRecorder`, delegates to the mux, and logs via `logRequest`.
- `logging.go` centralises all structured logging helpers (`logRequest`, `logTask`, `logTaskLaunchStage`, `logDockerSandboxEvent`, `logCustomArgConflict`) and suppresses noisy UI poll GETs (`/events`, `/transcript`, `/timeline`, bare task detail) so only errors surface. Uses `adapters.Redact` to scrub secrets before printing.
- `runtime_activity.go` computes per-task `RuntimeActivity` from three sources — bound provider session health, harness ownership, and durable task status — returning one of `live/offline/orphaned/unknown` plus optional `TurnActivity`. `reconcileRuntimeActivity` mutates Task state offline→failed or orphaned→interrupted (emitting `EventKindLifecycle` events) while skipping mutation during an in-flight control op protected by `controlMu`/`activeControls`. `attachRuntimeActivity` composes compute+reconcile for callers.

Dependency direction is inward-only: this module depends on sibling packages (`internal/project`, `internal/task`, `internal/runtime`, `internal/blackboardv2`, `internal/projectinterface`, `internal/preflight`, `internal/adapters`, `internal/webfs`) but exposes no internal interfaces beyond `Server`/`Config`/`ProviderSessionFactory`.

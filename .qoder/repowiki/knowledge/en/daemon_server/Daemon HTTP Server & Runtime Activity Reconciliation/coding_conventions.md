- Every HTTP handler decodes JSON into a local struct, branches on typed error sentinel values via `errors.Is(err, <pkg>.ErrXxx)`, and returns `writeError(response, code, message)` — never writes raw strings directly to the response.
- Optional fields in PATCH/update handlers are modelled as pointer-typed JSON fields (`*string`, `*runtimeprofile.Provider`, `*runtimeprofile.Fields`) and callers fall back to re-fetching the existing entity when a field is nil.
- Cross-cutting concerns (auth, CORS preflight, origin validation, request timing) are implemented as thin wrappers around `http.ResponseWriter` (`statusRecorder`) and helper functions (`authorized`, `publicPath`, `allowedOrigin`) invoked once at the top of `ServeHTTP` rather than per-handler middleware.
- Secret-bearing data is never logged verbatim: sensitive values go through `adapters.Redact` or are extracted from already-redacted typed errors (`CustomArgConflictError.Argument`) before being printed.
- Liveness transitions are expressed as small named constants (`runtimeLivenessLive`, `runtimeLivenessOffline`, `runtimeLivenessOrphaned`, `runtimeLivenessUnknown`, `runtimeTurnBusy`, `runtimeTurnIdle`) and applied uniformly by `computeRuntimeActivity` + `reconcileRuntimeActivity` instead of ad-hoc checks in handlers.

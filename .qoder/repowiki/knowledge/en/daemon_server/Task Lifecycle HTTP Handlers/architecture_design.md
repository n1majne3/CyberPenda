Three files in package `daemon` implement the HTTP-facing surface of task lifecycle on `*Server`.

- `task_handlers.go` is the core: each exported `handle*` method is an HTTP handler (CRUD via `handleCreateTask`, `handleListTasks`, `handleGetTask`, `handleDeleteTask`; control via `handleStopTask`, `handleFinishTask`, `handleResumeTask`; steering via `handleQueueSteerTask`, `handleSteerTask`, `handleProviderSessionSteer`, `handleProviderPermissionResponse`; observability via `handleTaskEvents`, `handleTaskTimeline`, `handleTaskTranscript`; continuation via `handleTaskContinuation`).
- Launch flow is layered through a `taskLaunchPlan` struct that carries the resolved profile, materialized credentials, projected runtime config, adapter, stop confirmation, and Blackboard v2 metadata. Builders (`buildTaskLaunchPlan`, `prepareBlackboardV2TaskLaunchPlan`, `buildTaskLaunchPlanWithBinding`) separate plan construction from execution; `launchTaskInBackground` then opens a persistent provider session (via `providerSessionFactory`) when available, otherwise falls back to a one-shot `runtime.Adapter`, and finally delegates to `server.harness.Launch` in a goroutine.
- Blackboard v2 continuity is gated by `blackboardV2Continuity.CreateContinuation` with `Precommit`/`BindGrant`/`UnbindGrant` callbacks that re-project the launch plan against a fixed global model-provider snapshot so SQLite transactions never re-enter `modelProviders.Service`.
- `launch_handlers.go` exposes a single non-task endpoint `handleResolveLaunchRuntimeProfile` that resolves a provider/model selection into a concrete profile before task creation.
- `test_helpers.go` exposes `CreateLocalRuntimeProfile` as a test-only shortcut to bypass HTTP round-trips when setting up fixtures.

Dependency direction: handlers depend outward on `project`, `profiles`, `tasks`, `skills`, `modelprovider`, `preflight`, `runner`, `runtime`, `runtimeprofile`, `transcript`, `timeline`, `blackboardv2`, and `adapters`; nothing inside this module imports the HTTP router or mux — it only defines methods on `*Server`.

Two independent HTTP transport layers live in package `daemon`:

- Skill registry (`skill_handlers.go`): exposes `/api/skills/*` endpoints backed by `server.skills` (a `skill.SkillStore` interface from `pentest/internal/skill`). Handlers are thin request/response adapters that decode JSON into `skillWriteRequest`/`skill.ImportRequest`, call store methods like `Publish`, `Import`, `Delete`, `EnabledSkills`, and map domain errors via `writeSkillError` to HTTP status codes. Responses are shaped through `newSkillResponse`, which redacts sensitive fields: builtin `SourceProvenance` details are stripped to just `kind="builtin"`, and `UPSTREAM.md` is filtered from file listings.

- Blackboard v2 transport (`blackboard_v2_http.go`): registers routes under `/api/v2/projects/{id}/blackboard/*` plus report endpoints. Every handler follows a uniform pipeline — `authenticateBlackboardV2` resolves either an operator principal (via `server.authToken`) or a Continuation grant (via `server.projectInterfaceGrants.Resolve`), then dispatches through `serveBlackboardV2` / `serveBlackboardV2Conditional` / `serveBlackboardV2Result`, which bind a `ContinuationAuthority`, optionally claim/capture trusted synchronization via `Idempotency-Key` fingerprints, invoke the action, and wrap results in `writeBlackboardV2Success`/`writeBlackboardV2Error`. Errors are normalized through `asBlackboardV2Error` and mapped to HTTP codes by `blackboardV2HTTPStatus`; conditional GETs use revision ETags with honest `If-None-Match` handling. Input bodies are strictly validated via `decodeBlackboardV2JSON` (4 MiB limit, unknown-field rejection).

Dependency direction: both files depend only on `pentest/internal/skill` and `pentest/internal/blackboardv2` respectively; they never reach into persistence directly, delegating all state mutations to injected interfaces on `Server`.

- HTTP handlers follow a three-step pattern: authenticate/authorize, validate+decode body, delegate to a service method, then write a typed response — error paths always go through a dedicated `write*Error` helper rather than direct `http.Error` calls.
- Domain-layer errors are surfaced as typed sentinel errors (`skill.ErrInvalidSkill`, `skill.ErrNotFound`, `skill.ErrEnabled`, `blackboardv2.Error`) and translated to HTTP status codes in a single switch/mapper function instead of per-handler checks.
- Responses are produced through small adapter functions (`newSkillResponse`, `publicSkillSource`, `publicSkillFiles`) that enforce field-level redaction rules (hiding builtin provenance details and `UPSTREAM.md`) before serialization.
- Blackboard v2 handlers share a common execution wrapper (`serveBlackboardV2` → `serveBlackboardV2Result`) that centralizes Continuation binding, synchronization fingerprinting, and sync attachment, so individual handlers only supply the business action closure.

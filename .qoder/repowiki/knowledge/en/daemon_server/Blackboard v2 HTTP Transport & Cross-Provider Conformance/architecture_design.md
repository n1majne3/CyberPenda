Single-file HTTP adapter (`blackboard_v2_http.go`) in package `daemon` that mounts routes under `/api/v2/projects/{id}/blackboard/*` via Go 1.22+ path parameters. Each handler follows a uniform pipeline: `authenticateBlackboardV2` (operator token or Continuation grant resolution), optional `requireBlackboardV2IdempotencyKey`, JSON decode with `DisallowUnknownFields` and 4 MiB limit, then dispatch through `serveBlackboardV2` / `serveBlackboardV2Conditional` which bind a `ContinuationAuthority`, optionally claim/capture trusted synchronization attachments keyed by an Idempotency-Key fingerprint, invoke the domain action on `server.blackboardV2`, and marshal either a plain result or a `{..., "sync": ...}` envelope with ETag/If-None-Match conditional support. Mutating endpoints (`changes`, `evidence:retain`, `attempts/{action}`, `continuation:finish`) pass `requireLive=false` so exact replays survive Finish/supersession; read-only endpoints (`snapshot`, `health`, `records/{key}`, `history`) pass `requireLive=true`. Error mapping normalizes storage/busy errors to `storage_busy`/503 and strips sync from auth/internal failures. The two test files are leaf contracts: `blackboard_v2_boundary_test.go` asserts zero legacy v1 route exposure and exactly six trusted MCP tools, while `blackboard_v2_claude_pi_conformance_test.go` drives real provider launches to verify shared snapshot bytes, compact five-field launch header, single checklist projection, no identity leakage into argv/env, and restart-resume pin freshness across Claude Code and Pi.
